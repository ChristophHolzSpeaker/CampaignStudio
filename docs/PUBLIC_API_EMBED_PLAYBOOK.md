# Public API And Embed Playbook

This playbook describes how to test CampaignStudio from a separate external project. The target use case is an interface with campaign navigation on the left, a tokenized landing page iframe in the center, and an AI/chat workflow on the right.

## Required CampaignStudio Environment

Set these values in the CampaignStudio app:

```bash
CAMPAIGN_WRITE_API_TOKEN="your-write-token"
LEAD_READ_API_TOKEN="your-read-token"
EMBED_PREVIEW_TOKEN_SECRET="long-random-secret"
```

For the MVP:

- `GET /api/public/v1/campaigns` accepts either the read or write token.
- `POST /api/public/v1/campaigns` requires the write token.
- `/embed/[slug]` requires the signed `?token=...` generated by the API.

## API Base URL

Local example:

```ts
const CAMPAIGNSTUDIO_BASE_URL = 'http://localhost:5173';
```

Production example:

```ts
const CAMPAIGNSTUDIO_BASE_URL = 'https://your-campaignstudio-domain.com';
```

## Authentication

All public API calls require bearer auth:

```ts
const headers = {
	Authorization: `Bearer ${process.env.CAMPAIGNSTUDIO_API_TOKEN}`,
	'Content-Type': 'application/json'
};
```

Do not expose API tokens in browser code. In an external test project, call CampaignStudio from your own server route or API handler.

## OpenAPI Docs

CampaignStudio exposes OpenAPI at:

```http
GET /api/public/v1/openapi.json
```

Example:

```bash
curl \
	-H "Authorization: Bearer $CAMPAIGNSTUDIO_API_TOKEN" \
	http://localhost:5173/api/public/v1/openapi.json
```

The OpenAPI document includes:

- `GET /api/public/v1/page-sections/schema`
- `GET /api/public/v1/campaigns`
- `POST /api/public/v1/campaigns`
- Lead journey endpoints

## Discover Landing Page Section Schema

Use this before creating pages from an external agent or Claude workflow:

```http
GET /api/public/v1/page-sections/schema
```

Example:

```bash
curl \
	-H "Authorization: Bearer $CAMPAIGNSTUDIO_API_TOKEN" \
	http://localhost:5173/api/public/v1/page-sections/schema
```

The response includes:

- Landing page document schema
- Allowed section types
- Section-specific prop schemas
- Guidance for when to use each section

## Render The Iframe Preview

Minimal iframe:

```html
<iframe
	id="campaign-preview"
	src="http://localhost:5173/embed/landing-page-c1-v1?token=..."
	style="width: 100%; border: 0;"
></iframe>
```

The embed route returns `404` when:

- The token is missing.
- The token is invalid.
- The slug does not exist.
- `EMBED_PREVIEW_TOKEN_SECRET` is not configured.

## Auto-Resize The Iframe

CampaignStudio posts height messages to the parent window.

Message shape:

```ts
type CampaignStudioEmbedHeightMessage = {
	type: 'campaignstudio:embed-height';
	height: number;
	slug: string;
	campaignId: number | null;
	campaignPageId: number | null;
};
```

Parent-side vanilla JavaScript:

```html
<iframe id="campaign-preview" src="..." style="width: 100%; border: 0;"></iframe>

<script>
	const iframe = document.getElementById('campaign-preview');

	window.addEventListener('message', (event) => {
		if (event.data?.type !== 'campaignstudio:embed-height') return;

		iframe.style.height = `${event.data.height}px`;
	});
</script>
```

Recommended production hardening:

```js
const allowedOrigin = 'https://your-campaignstudio-domain.com';

window.addEventListener('message', (event) => {
	if (event.origin !== allowedOrigin) return;
	if (event.data?.type !== 'campaignstudio:embed-height') return;

	iframe.style.height = `${event.data.height}px`;
});
```

For local development, use:

```js
const allowedOrigin = 'http://localhost:5173';
```

## Create A Campaign

Use this endpoint to create a draft campaign and landing page from structured JSON.

```http
POST /api/public/v1/campaigns
```

Example:

```bash
curl \
	-X POST \
	-H "Authorization: Bearer $CAMPAIGNSTUDIO_WRITE_TOKEN" \
	-H "Content-Type: application/json" \
	http://localhost:5173/api/public/v1/campaigns \
	-d '{
		"campaign": {
			"name": "External Test Campaign",
			"audience": "Conference organizers",
			"format": "Keynote",
			"topic": "AI transformation",
			"language": "English",
			"geography": "Europe",
			"notes": "Created from external interface"
		},
		"content_json": {
			"version": 1,
			"title": "External Test Landing Page",
			"slug": "external-test-landing-page",
			"sections": []
		},
		"change_note": "Created from external testing project"
	}'
```

The `sections` array above is only illustrative. Real usage must provide valid sections. Fetch `/api/public/v1/page-sections/schema` first and generate a valid landing page document from that schema.

Create response shape:

```json
{
	"ok": true,
	"data": {
		"campaignId": 1,
		"campaignPageId": 10,
		"pageSlug": "external-test-landing-page-c1-v1",
		"campaignUrl": "/campaigns/1",
		"previewUrl": "/campaigns/1/landing-page",
		"embedUrl": "http://localhost:5173/embed/external-test-landing-page-c1-v1?token=..."
	}
}
```

## Minimal Server Proxy Example

Use a server endpoint in the external project so API tokens stay private.

```ts
export async function getCampaignNav() {
	const response = await fetch(`${CAMPAIGNSTUDIO_BASE_URL}/api/public/v1/campaigns`, {
		headers: {
			Authorization: `Bearer ${process.env.CAMPAIGNSTUDIO_API_TOKEN}`
		}
	});

	if (!response.ok) {
		throw new Error(`CampaignStudio returned ${response.status}`);
	}

	return response.json();
}
```

Browser code should call your external app, not CampaignStudio directly.

## MVP Testing Checklist

1. Start CampaignStudio locally.
2. Ensure `EMBED_PREVIEW_TOKEN_SECRET` is set.
3. Ensure `CAMPAIGN_WRITE_API_TOKEN` and `LEAD_READ_API_TOKEN` are set.
4. Call `GET /api/public/v1/campaigns`.
5. Copy one `embedUrl` into an iframe.
6. Confirm missing token returns `404`.
7. Confirm valid `embedUrl` renders.
8. Confirm parent receives `campaignstudio:embed-height`.
9. Confirm iframe resizes when content height changes.
10. Fetch `/api/public/v1/openapi.json` and provide it to Claude as API context.
